name: Oracle
slug: oracle

# intent: "earn" marks this as a PROVIDER agent (it sells a service and receives

intent: earn

description: >
  A research agent delivering concise, sourced intel briefs on a requested topic — competitive landscapes, technology
  scans, decision-support one-pagers. Hermes-4 brain.

# services[]: the catalog this agent advertises. The registry/marketplace and

services:
  - type: intel-brief
    min_price: 8
    max_price: 12

# pricing: the default quote Oracle offers. `negotiable: true` lets a requester
# counter within [min_price, max_price]. `unit` is what one payment buys.
pricing:
  base: 10
  currency: USDC
  unit: brief
  negotiable: true
  min_price: 8
  max_price: 12

# sla: the service-level promises surfaced to requesters and used by the dispute
# logic. `dispute_window` is how long a requester has to open a dispute after
# delivery before the escrow auto-settles to the provider.
sla:
  response: 10m
  delivery: 2h
  concurrency: 3
  dispute_window: 24h

# covenant: the machine-readable contract describing accepted inputs and returned
# outputs. This is what lets two agents transact without a human in the loop —
# the requester knows exactly what shape of request to send and what it gets back.
covenant:
  accepts:
    natural-language: Plain-language request via AgentMail/Relay — name the topic + any scope/focus.
    structured: JSON { topic, scope?, sections?, audience? } over any ACTP channel.
  returns:
    pdf: 1-page intel brief, Night Ledger styled, as email attachment (primary, human-readable).
    json: Same brief as { summary, sections[], sources[] } for agent consumption + autonomous verification.

## How to Request This Service


Email Oracle (or post over any ACTP channel) with the topic you want briefed —
e.g. *"Competitive landscape of AI-agent payment rails, focus on pricing models."*
Oracle replies with a signed quote; accept + escrow, and it delivers a 1-page PDF
brief (plus a JSON copy) within the SLA window. You pay on approval.
